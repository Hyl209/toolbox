# NCM GUI 设计说明

目标：基于现有 ncm_to_mp3.py 的转换能力，新增一个 PySide6 图形界面程序，支持拖拽多个 .ncm 文件或文件夹，指定输出目录后执行批量转换。

方案对比：
1. 直接在 GUI 里重写转换逻辑
   - 缺点：与命令行重复，后期难维护
2. GUI 直接调用命令行子进程
   - 优点：复用现有入口
   - 缺点：参数/输出解析略绕，测试不够细
3. 抽出共享核心 + CLI/GUI 双入口（推荐）
   - 优点：逻辑复用最好，测试粒度更细，后期加 exe/更多界面更方便

推荐方案：
- 把文件收集与单文件转换逻辑保留在可导入模块中
- CLI 继续保留
- 新增 GUI 文件调用共享函数

GUI 结构：
1. 主窗口
   - 拖拽提示区
   - 待转换文件列表
   - 输出目录输入框 + 选择按钮
   - 覆盖选项复选框
   - 开始转换按钮
   - 进度条
   - 日志文本框
2. 拖拽行为
   - 接收多个文件/文件夹
   - 文件夹自动递归扫描 .ncm
   - 自动去重
3. 转换行为
   - 必须先指定输出目录
   - 批量逐个转换
   - 每完成一个更新日志与进度
   - 失败项保留错误信息
4. 输出目录记忆
   - 用 QSettings 保存上次选择的输出目录
   - 下次启动自动恢复
   - 用户可随时重新选择并覆盖旧路径
   - 如果保存的路径已失效，界面提示并允许重新选择
5. 代码组织
   - ncm_to_mp3.py：共享逻辑 + CLI
   - ncm_gui.py：PySide6 界面层
   - tests_ncm_to_mp3.py：补共享逻辑测试
   - tests_ncm_gui.py：补基础 GUI/拖拽收集测试（尽量无界面阻塞）

验证：
- 共享逻辑测试通过
- GUI 基础测试通过
- 用样本目录手动验证：能添加 .ncm、指定输出目录、成功生成 mp3
